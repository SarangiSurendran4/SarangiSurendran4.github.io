Release Docs  ·  Early Access Testing

Diagnostics: Restructuring docs around real debugging workflows

Diagnostics was a complex debugging feature. Technically correct documentation risked being practically unhelpful if users could not map it to what they were actually experiencing.

Onboarded as an early-access tester, reproduced real user scenarios, and listened to customer calls to understand how users think when something breaks. Restructured the guide around that workflow rather than the feature's component list.

Added a dedicated troubleshooting section, cross-linked it from every related error state, and created a support video following the same task-based flow.

The PM noted the documentation reflected the customer's thought process. The article became a team reference and the feature saw strong post-launch adoption signals.

PM appreciation for customer-voice docs. Strong adoption post-launch.

Usability Research  ·  DAP  ·  Content Strategy

Adoption: Turning documentation into a measurable engagement driver

A new feature had low adoption despite documentation existing. Users could read the docs and still not know what to do with the feature in their own context.

Led usability testing with Digital Adoption Consultants. They confirmed support articles were too abstract. Users needed to see the feature applied to situations they actually recognised.

Rewrote community posts around real user scenarios and surfaced them via a Whatfix Launcher on the support site, giving users a direct path from documentation to practical application.

Community traffic increased after the Launcher went live. Feature feedback initiatives received 100+ responses and feature usage signals improved measurably.

100+ feedback responses. Measurable increase in feature engagement.

OKR delivery: Giving a PM their quarterly data in one day, not one month

A PM needed QuickRead engagement data to close a Q1 OKR. Without it, the plan was to instrument new tracking and wait a full month for data to accumulate.

Checked whether Whatfix auto-capture had been passively recording the relevant interaction data in the background before any manual tracking was set up.

Used the existing auto-captured historical data to surface exactly what the PM needed. No new instrumentation, no waiting.

The PM received the data the same day the request came in. The Q1 OKR was closed on schedule, a month ahead of the original plan.

UX Writing  ·  Content Governance

UX audit: Fixing product-wide terminology drift before it reached users

UX copy had drifted across 7+ pods over multiple release cycles. Inconsistent terminology was creating confusion at key decision points in the product.

Audited microcopy across all affected pods and built a tracking sheet mapping each inconsistency to its location and a proposed correction.

Collaborated with the UX writer to resolve every flagged item and formalise the corrections into updated guidelines applying across all pods going forward.

Consistent product terminology across 7+ pods. The guidelines now serve as the single reference for all future UX copy decisions.
